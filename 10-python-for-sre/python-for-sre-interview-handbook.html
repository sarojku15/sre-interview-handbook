Chapter 1

Python Fundamentals

Interviewers use this chapter to confirm baseline fluency before moving into automation scenarios. Expect quick, rapid-fire questions rather than deep ones — the bar is "can you write correct Python without hesitation," not depth.

Core Concepts

name = "Saroj"       # str
age = 11              # int
years_exp = 11.5       # float
is_oncall = True      # bool

print(f"{name} has {years_exp} years of experience")

# Type conversion
str(42); int("42"); float("3.14"); bool(0)   # bool(0) is False, bool(1) is True

# Multiple assignment
host, port = "prod-db-01", 5432

# f-strings (preferred over % or .format() in modern interviews)
print(f"Connecting to {host}:{port}")

Question 1 — Is Python statically or dynamically typed?

Answer

Dynamically typed: a variable's type is determined at runtime and can change, e.g. x = 5 then x = "five" is legal. Python is also strongly typed — it won't silently coerce "5" + 5; that raises a TypeError.

Question 2 — Mutable vs immutable types?

Answer

Immutable: int, float, str, tuple, frozenset, bool. Mutable: list, dict, set. This matters operationally — passing a mutable default argument, or mutating a list while iterating it, are two of the most common production bugs.

# Classic mutable-default-argument bug
def add_host(host, pool=[]):     # BUG: default list is shared across calls
    pool.append(host)
    return pool

# Fix
def add_host(host, pool=None):
    if pool is None:
        pool = []
    pool.append(host)
    return pool

Question 3 — is vs ==?

Answer

== compares values; is compares object identity (same memory reference). Use is for None checks (if x is None), and == for everything else. if x == None works but is considered non-idiomatic.

Practice Exercise

Task: Write a script that stores a service name, port, and a list of environment tags, then prints a one-line summary using an f-string.

service = "checkout-api"
port = 8080
tags = ["prod", "payments", "tier-1"]
print(f"{service} listens on :{port} [{', '.join(tags)}]")
# checkout-api listens on :8080 [prod, payments, tier-1]
Common mistake: using mutable default arguments (def f(x=[])), which persist across calls and cause hard-to-trace state leaks — a favorite trick question.
Interview tip: if asked "is Python interpreted or compiled," the accurate answer is both: source compiles to bytecode (.pyc) which the CPython VM interprets. Saying "purely interpreted" is a minor red flag to strong interviewers.
Chapter 2

Conditional Statements

Simple on the surface, but interviewers use these to see if you write clean, readable branching logic instead of nested pyramids — a real production-readability signal.

Core Concepts

status_code = 503

if status_code < 300:
    print("OK")
elif status_code < 500:
    print("Client error")
else:
    print("Server error")

# Ternary (ideal for short assignments)
health = "healthy" if status_code < 300 else "unhealthy"

# Truthiness: 0, "", [], {}, None, False are all falsy
if not error_list:
    print("No errors")

Question 1 — Check even or odd

def is_even(n):
    return n % 2 == 0

print(is_even(17))   # False

Question 2 — Largest of three numbers

def largest(a, b, c):
    return max(a, b, c)   # built-in max() beats manual if/elif in interviews

print(largest(4, 19, 7))   # 19

Question 3 — Leap year check

def is_leap(year):
    return year % 4 == 0 and (year % 100 != 0 or year % 400 == 0)

print(is_leap(2024), is_leap(1900), is_leap(2000))  # True False True

Practice Exercise

Task: Given an HTTP status code, classify it as Informational / Success / Redirect / Client Error / Server Error using a dict-based lookup instead of a long if/elif chain (a pattern interviewers like to see for scalability).

def classify(code):
    ranges = {1: "Informational", 2: "Success", 3: "Redirect",
              4: "Client Error", 5: "Server Error"}
    return ranges.get(code // 100, "Unknown")

print(classify(404), classify(200), classify(503))
# Client Error Success Server Error
Common mistake: deeply nested if/elif/else pyramids instead of early returns or dict-based dispatch — interviewers reading this as a readability/maintainability red flag for production code.
Interview tip: mention guard clauses (if not valid: return early) as your default style — it signals you write for the next reader, which matters heavily in on-call codebases.
Chapter 3

Loops

Loops are the backbone of every automation script — iterating over log lines, pods, hosts, files. Interviewers watch for enumerate(), list comprehensions, and correct use of break/continue as fluency signals.

Core Concepts

for i in range(5):
    print(i)              # 0 1 2 3 4

hosts = ["web-1", "web-2", "web-3"]
for idx, host in enumerate(hosts, start=1):
    print(f"{idx}: {host}")

i = 0
while i < 3:
    i += 1
    if i == 2:
        continue           # skip this iteration
    print(i)

for host in hosts:
    if host == "web-2":
        break               # stop the loop entirely
    print(host)

Question 1 — Multiplication table

n = 7
for i in range(1, 11):
    print(f"{n} x {i} = {n*i}")

Question 2 — Fibonacci sequence

def fibonacci(n):
    a, b = 0, 1
    result = []
    for _ in range(n):
        result.append(a)
        a, b = b, a + b
    return result

print(fibonacci(8))  # [0, 1, 1, 2, 3, 5, 8, 13]

Question 3 — Check for prime numbers

def is_prime(n):
    if n < 2:
        return False
    for i in range(2, int(n ** 0.5) + 1):
        if n % i == 0:
            return False
    return True

print([x for x in range(2, 30) if is_prime(x)])

Question 4 — Factorial

def factorial(n):
    result = 1
    for i in range(2, n + 1):
        result *= i
    return result

print(factorial(6))  # 720

Practice Exercise

Task: Given a list of pod restart counts, use a list comprehension to find pods restarting more than 5 times.

pods = {"api-1": 2, "api-2": 9, "worker-1": 12, "worker-2": 0}
flapping = [name for name, restarts in pods.items() if restarts > 5]
print(flapping)  # ['api-2', 'worker-1']
Common mistake: mutating a list while iterating over it (for x in items: items.remove(x)) — silently skips elements. Iterate over a copy (items[:]) or build a new list instead.
Interview tip: when the task is "transform every item," reach for a list comprehension by default — interviewers read this as idiomatic Python, not just working Python.
Chapter 4

Functions

Functions are where interviewers check whether you understand scope, argument passing, and when to reach for a lambda versus a named function — all of which show up constantly in automation scripts and Kubernetes/AWS SDK callbacks.

Core Concepts

def restart_service(name, retries=3, *, dry_run=False):
    # Keyword-only dry_run forces callers to be explicit.
    for attempt in range(1, retries + 1):
        if dry_run:
            print(f"[dry-run] would restart {name} (attempt {attempt})")
        else:
            print(f"restarting {name} (attempt {attempt})")

restart_service("nginx", dry_run=True)

# *args / **kwargs
def run_command(cmd, *args, **kwargs):
    print(cmd, args, kwargs)

run_command("kubectl", "get", "pods", namespace="prod")
# kubectl ('get', 'pods') {'namespace': 'prod'}

# Lambda — short, single-expression only
square = lambda x: x ** 2
sorted_hosts = sorted(["web-10", "web-2", "web-1"], key=lambda h: int(h.split("-")[1]))

Question 1 — Default vs keyword arguments?

Answer

Default arguments supply a fallback value if the caller omits it (def f(x=5)). Keyword arguments are supplied by name at the call site (f(x=5)), independent of position. You can force keyword-only arguments with a bare * in the signature — useful for flags like dry_run where positional use would be error-prone.

Question 2 — What is a closure, and where would you use one in SRE tooling?

Answer

A closure is a function that remembers variables from its enclosing scope even after that scope has exited. Common use: building a retry decorator that "remembers" a configured max-retries value.

def make_retrier(max_attempts):
    def retrier(func):
        def wrapper(*args, **kwargs):
            for attempt in range(1, max_attempts + 1):
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    print(f"attempt {attempt} failed: {e}")
            raise RuntimeError("all retries exhausted")
        return wrapper
    return retrier

@make_retrier(3)
def call_api():
    ...

Question 3 — Recursion example: sum of digits

def sum_digits(n):
    if n < 10:
        return n
    return n % 10 + sum_digits(n // 10)

print(sum_digits(9875))  # 29

Practice Exercise

Task: Write a simple calculator function that takes an operator string and two numbers, dispatching via a dict instead of if/elif.

import operator

OPS = {"+": operator.add, "-": operator.sub, "*": operator.mul, "/": operator.truediv}

def calculate(op, a, b):
    if op not in OPS:
        raise ValueError(f"unsupported operator: {op}")
    return OPS[op](a, b)

print(calculate("*", 6, 7))  # 42
Common mistake: forgetting that Python passes object references — mutating a mutable argument (a list/dict) inside a function mutates the caller's object too, unless you explicitly copy it first.
Interview tip: if asked to write a "calculator" or "dispatcher," using a dict of callables instead of if/elif signals you think about extensibility — a small but noticeable senior-level signal.
Chapter 5

Strings

String manipulation questions are near-universal in SRE interviews because log parsing, config templating, and text processing all live here. Expect at least two or three of these in any Python round.

Question 1 — Reverse a string

def reverse(s):
    return s[::-1]

print(reverse("kubernetes"))  # setenrebuk

Question 2 — Palindrome check

def is_palindrome(s):
    s = s.lower().replace(" ", "")
    return s == s[::-1]

print(is_palindrome("A man a plan a canal Panama"))  # True

Question 3 — Anagram check

def is_anagram(a, b):
    return sorted(a.lower()) == sorted(b.lower())

print(is_anagram("listen", "silent"))  # True

Question 4 — Count vowels

def count_vowels(s):
    return sum(1 for ch in s.lower() if ch in "aeiou")

print(count_vowels("Site Reliability Engineering"))  # 11

Question 5 — Count words

def word_count(s):
    return len(s.split())

print(word_count("kubectl get pods -n prod"))  # 5

Question 6 — First non-repeating character

def first_unique(s):
    for ch in s:
        if s.count(ch) == 1:
            return ch
    return None

print(first_unique("swiss"))  # w

Question 7 — Remove duplicate characters (preserve order)

def remove_dupes(s):
    seen = set()
    result = []
    for ch in s:
        if ch not in seen:
            seen.add(ch)
            result.append(ch)
    return "".join(result)

print(remove_dupes("mississippi"))  # misp

Question 8 — String compression

def compress(s):
    if not s:
        return s
    result = []
    count = 1
    for i in range(1, len(s) + 1):
        if i < len(s) and s[i] == s[i - 1]:
            count += 1
        else:
            result.append(s[i - 1] + (str(count) if count > 1 else ""))
            count = 1
    compressed = "".join(result)
    return compressed if len(compressed) < len(s) else s

print(compress("aaabbbccd"))  # a3b3c2d

Question 9 — Character frequency

from collections import Counter

def char_frequency(s):
    return dict(Counter(s.replace(" ", "")))

print(char_frequency("error error warn"))
# {'e': 2, 'r': 6, 'o': 2, 'w': 1, 'a': 1, 'n': 1}

Practice Exercise

Task: Given a raw log line, extract just the log level (INFO/WARN/ERROR) using string methods — no regex.

line = "2026-07-07 10:12:03 ERROR failed to connect to db"

def extract_level(line):
    for level in ("ERROR", "WARN", "INFO"):
        if level in line:
            return level
    return "UNKNOWN"

print(extract_level(line))  # ERROR
Common mistake: using += to build a string inside a large loop. Each concatenation creates a new string object (strings are immutable), which is O(n²) for n appends. Use "".join(list_of_parts) instead.
Interview tip: collections.Counter is the fastest way to answer any "count frequency of X" question — naming it unprompted is a strong signal you know the standard library, not just the language.
Chapter 6

Lists

Lists back most SRE data wrangling — host inventories, pod lists, metric samples. Interviewers look for comfort with slicing, comprehensions, and knowing when not to use a list (e.g. membership testing is O(n) on a list vs O(1) on a set).

Question 1 — Find the largest and smallest number

nums = [45, 3, 89, 12, 67]
print(max(nums), min(nums))  # 89 3

Question 2 — Remove duplicates (order doesn't matter)

nums = [1, 2, 2, 3, 3, 3, 4]
unique = list(set(nums))
print(sorted(unique))  # [1, 2, 3, 4]

Question 3 — Find duplicates

from collections import Counter

nums = [1, 2, 2, 3, 3, 3, 4]
dupes = [n for n, c in Counter(nums).items() if c > 1]
print(dupes)  # [2, 3]

Question 4 — Second largest number

def second_largest(nums):
    uniq = sorted(set(nums), reverse=True)
    return uniq[1] if len(uniq) > 1 else None

print(second_largest([45, 3, 89, 12, 89]))  # 45

Question 5 — Rotate an array by k positions

def rotate(arr, k):
    k %= len(arr)
    return arr[-k:] + arr[:-k]

print(rotate([1, 2, 3, 4, 5], 2))  # [4, 5, 1, 2, 3]

Question 6 — Merge two sorted lists

def merge_sorted(a, b):
    return sorted(a + b)

print(merge_sorted([1, 3, 5], [2, 4, 6]))  # [1, 2, 3, 4, 5, 6]

Question 7 — Sort without using sort() (bubble sort)

def bubble_sort(arr):
    arr = arr[:]
    n = len(arr)
    for i in range(n):
        for j in range(n - i - 1):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
    return arr

print(bubble_sort([5, 1, 4, 2, 8]))  # [1, 2, 4, 5, 8]

Question 8 — Common elements between two lists

a = ["web-1", "web-2", "web-3"]
b = ["web-2", "web-3", "web-4"]
print(list(set(a) & set(b)))  # ['web-2', 'web-3'] (order not guaranteed)

Practice Exercise

Task: Given CPU utilization samples for a host, find how many exceeded an 80% threshold, and the average.

samples = [45, 82, 91, 76, 88, 33, 95]
threshold = 80
over = [s for s in samples if s > threshold]
print(len(over), sum(samples) / len(samples))
# 4 72.85714285714286
Common mistake: using in to check membership repeatedly on a large list inside a loop — O(n) per check. Convert to a set first if you're checking membership many times; that's O(1) per check.
Interview tip: know the difference between list.sort() (in-place, returns None) and sorted(list) (returns a new list). Returning None from a chained .sort() call is a classic bug interviewers plant on purpose.

Dictionary

Dictionaries are the workhorse of config parsing, JSON handling, and building lookup tables from log/metric data — expect heavy use across the automation chapters that follow.

Question 1 — Word frequency count

from collections import Counter

text = "error timeout error connection error timeout"
freq = Counter(text.split())
print(freq)
# Counter({'error': 3, 'timeout': 2, 'connection': 1})
print(freq.most_common(1))  # [('error', 3)]

Question 2 — Character frequency (dict comprehension)

s = "kubernetes"
freq = {ch: s.count(ch) for ch in set(s)}
print(freq)

Question 3 — Merge dictionaries

defaults = {"timeout": 30, "retries": 3}
overrides = {"retries": 5, "region": "ap-south-1"}

merged = {**defaults, **overrides}   # Python 3.5+
print(merged)  # {'timeout': 30, 'retries': 5, 'region': 'ap-south-1'}

# Python 3.9+
merged2 = defaults | overrides

Question 4 — Invert a dictionary

status_codes = {"OK": 200, "NOT_FOUND": 404, "SERVER_ERROR": 500}
inverted = {v: k for k, v in status_codes.items()}
print(inverted)  # {200: 'OK', 404: 'NOT_FOUND', 500: 'SERVER_ERROR'}

Question 5 — Group items by key

pods = [
    {"name": "api-1", "node": "node-a"},
    {"name": "api-2", "node": "node-b"},
    {"name": "worker-1", "node": "node-a"},
]

from collections import defaultdict
by_node = defaultdict(list)
for pod in pods:
    by_node[pod["node"]].append(pod["name"])

print(dict(by_node))
# {'node-a': ['api-1', 'worker-1'], 'node-b': ['api-2']}

Practice Exercise

Task: Given a dict of service → response times (ms), find the service with the highest latency safely, without crashing on an empty dict.

latencies = {"checkout": 340, "search": 120, "auth": 890}

def slowest(latencies):
    if not latencies:
        return None
    return max(latencies, key=latencies.get)

print(slowest(latencies))  # auth
Common mistake: using dict[key] and letting a KeyError crash the script instead of dict.get(key, default) or checking membership first — a frequent source of production automation failures on missing config keys.
Interview tip: mention collections.defaultdict unprompted when grouping data by key — it removes the "if key not in dict: dict[key] = []" boilerplate and reads as idiomatic.
Chapter 8

Sets

Sets come up in interviews for exactly one reason: fast membership testing and clean set-algebra for comparing inventories — "which hosts are in Terraform but not actually running," "which pods are new since last deploy," etc.

Core Concepts

desired = {"web-1", "web-2", "web-3", "web-4"}
running = {"web-1", "web-2", "web-3", "web-5"}

missing = desired - running        # in desired but not running: {'web-4'}
unexpected = running - desired     # running but not desired: {'web-5'}
healthy = desired & running        # intersection: {'web-1','web-2','web-3'}
all_hosts = desired | running      # union
only_one_side = desired ^ running  # symmetric difference: {'web-4','web-5'}

Question 1 — Remove duplicates while ignoring order

ips = ["10.0.0.1", "10.0.0.2", "10.0.0.1", "10.0.0.3"]
unique_ips = set(ips)
print(unique_ips)

Question 2 — Union of two sets

team_a = {"alice", "bob"}
team_b = {"bob", "carol"}
print(team_a | team_b)  # {'alice', 'bob', 'carol'}

Question 3 — Intersection

print(team_a & team_b)  # {'bob'}

Question 4 — Difference

print(team_a - team_b)  # {'alice'}

Question 5 — Symmetric difference

print(team_a ^ team_b)  # {'alice', 'carol'}

Practice Exercise

Task: Reconcile a Terraform-managed host list against what's actually reporting into monitoring, and report drift in both directions.

terraform_hosts = {"db-1", "db-2", "db-3", "cache-1"}
monitored_hosts = {"db-1", "db-2", "cache-1", "cache-2"}

not_monitored = terraform_hosts - monitored_hosts
orphaned = monitored_hosts - terraform_hosts

print(f"Provisioned but not monitored: {not_monitored}")
print(f"Monitored but not in Terraform: {orphaned}")
# Provisioned but not monitored: {'db-3'}
# Monitored but not in Terraform: {'cache-2'}
Common mistake: forgetting sets are unordered and unindexable — my_set[0] raises TypeError. If you need order, use a list or a dict (which preserves insertion order in Python 3.7+).
Interview tip: when asked to compare "what's expected vs what's actually running" — a very common SRE framing — reach for set difference immediately. It's the cleanest, most senior-reading answer.
Chapter 9

File Handling

This is where SRE Python interviews diverge sharply from generic Python interviews. Expect a live-coding request to parse a log file — almost guaranteed in some form.

Core Concepts

# Always use a context manager — guarantees the file is closed even on exception
with open("app.log") as f:
    for line in f:              # streams line-by-line, doesn't load whole file into memory
        process(line.strip())

with open("report.txt", "w") as f:
    f.write("deploy complete\n")

with open("report.txt", "a") as f:
    f.write("post-deploy check passed\n")

Question 1 — Count ERROR lines in a log file

def count_errors(path):
    count = 0
    with open(path) as f:
        for line in f:
            if "ERROR" in line:
                count += 1
    return count

print(count_errors("app.log"))

Question 2 — Count WARN lines

def count_level(path, level):
    with open(path) as f:
        return sum(1 for line in f if level in line)

print(count_level("app.log", "WARN"))

Question 3 — Find the latest timestamp in a log

from datetime import datetime

def latest_timestamp(path):
    latest = None
    with open(path) as f:
        for line in f:
            ts_str = line[:19]   # "2026-07-07 10:12:03"
            try:
                ts = datetime.strptime(ts_str, "%Y-%m-%d %H:%M:%S")
            except ValueError:
                continue
            if latest is None or ts > latest:
                latest = ts
    return latest

print(latest_timestamp("app.log"))

Question 4 — Extract IP addresses from a log file

import re

def extract_ips(path):
    pattern = re.compile(r"\b(?:\d{1,3}\.){3}\d{1,3}\b")
    ips = []
    with open(path) as f:
        for line in f:
            ips.extend(pattern.findall(line))
    return ips

print(extract_ips("access.log"))

Question 5 — Read a CSV and write a summary report

import csv

def summarize_incidents(path):
    total, by_severity = 0, {}
    with open(path, newline="") as f:
        reader = csv.DictReader(f)
        for row in reader:
            total += 1
            sev = row["severity"]
            by_severity[sev] = by_severity.get(sev, 0) + 1
    return total, by_severity

total, by_sev = summarize_incidents("incidents.csv")
with open("summary.txt", "w") as f:
    f.write(f"Total incidents: {total}\n")
    for sev, count in by_sev.items():
        f.write(f"  {sev}: {count}\n")

Practice Exercise

Task: Write a function that safely reads a log file that may not exist, returning an empty list instead of crashing.

from pathlib import Path

def read_log_safe(path):
    p = Path(path)
    if not p.exists():
        print(f"warning: {path} not found, skipping")
        return []
    return p.read_text().splitlines()

lines = read_log_safe("maybe-missing.log")
Common mistake: using f.read() on a multi-gigabyte log file — loads the entire file into memory. Always iterate line-by-line (for line in f) for anything that could be large, which is the default assumption for production logs.
Interview tip: always open files with a with block. Forgetting it, or manually calling f.close(), reads as a junior habit — and if an exception is raised before close(), the file handle leaks.
Chapter 10

Exception Handling

SRE code runs unattended, often at 3 a.m. with nobody watching. Interviewers use this chapter to see if you handle failure explicitly instead of letting a script die silently or crash the whole pipeline over one bad record.

Core Concepts

try:
    value = int(input("Enter port: "))
except ValueError:
    print("Invalid input — must be an integer")
else:
    print(f"Port set to {value}")   # only runs if no exception occurred
finally:
    print("Input handling complete")  # always runs — cleanup goes here

# Raising your own exceptions with context
def get_config(cfg, key):
    if key not in cfg:
        raise KeyError(f"missing required config key: {key}")
    return cfg[key]

Question 1 — Handle divide-by-zero safely

def safe_divide(a, b):
    try:
        return a / b
    except ZeroDivisionError:
        return None

print(safe_divide(10, 0))  # None

Question 2 — Handle invalid user input in a loop

def get_int(prompt):
    while True:
        try:
            return int(input(prompt))
        except ValueError:
            print("Please enter a valid integer")

Question 3 — Handle file-not-found gracefully

def load_config(path):
    try:
        with open(path) as f:
            return f.read()
    except FileNotFoundError:
        print(f"config not found at {path}, using defaults")
        return None

Question 4 — Handle JSON parsing errors

import json

def parse_response(raw):
    try:
        return json.loads(raw)
    except json.JSONDecodeError as e:
        print(f"invalid JSON: {e}")
        return None

Question 5 — Catching specific exceptions vs bare except:?

Answer

Always catch the specific exception you expect (ValueError, KeyError, requests.Timeout). A bare except: also swallows KeyboardInterrupt and SystemExit, and hides bugs you didn't anticipate — a script that silently continues after an unexpected error is more dangerous in production than one that fails loudly.

Practice Exercise

Task: Write a function that calls an unreliable API and retries up to 3 times with a short delay, re-raising after the last failure.

import time

def call_with_retry(func, max_attempts=3, delay=2):
    last_error = None
    for attempt in range(1, max_attempts + 1):
        try:
            return func()
        except Exception as e:
            last_error = e
            print(f"attempt {attempt}/{max_attempts} failed: {e}")
            if attempt < max_attempts:
                time.sleep(delay)
    raise last_error
Common mistake: using a bare except: (or except Exception: pass) to make an error "go away." In an automation script this hides real failures — a deploy script that swallows every exception can report success while doing nothing.

def run_command(cmd):
    result = subprocess.run(cmd, capture_output=True, text=True)
    if result.returncode != 0:

Question 3 — Why is passing a list preferred over a shell string?

Answer

Avoid shell=True unless you specifically need shell features like pipes, and never combine it with unsanitized input.

Strings & Text (1–10)

 9. Title-case a sentence       s.title()

Two Sum & Classic Patterns (21–25)

21. Two Sum (return indices of pair summing to target)
def two_sum(nums, target):
    seen = {}
    for i, n in enumerate(nums):
        if target - n in seen:
            return [seen[target - n], i]

1. Kubernetes Pod Health Checker

#!/usr/bin/env python3
# Exits non-zero if any pod in the namespace is unhealthy.
import json, subprocess, sys, logging

logging.basicConfig(level=logging.INFO, format="%(asctime)s %(levelname)s %(message)s")

def get_pods(namespace):
    result = subprocess.run(
        ["kubectl", "get", "pods", "-n", namespace, "-o", "json"],
        capture_output=True, text=True
    )
    if result.returncode != 0:
        raise RuntimeError(f"kubectl failed: {result.stderr.strip()}")
    return json.loads(result.stdout)["items"]

def main(namespace="prod"):
    unhealthy = []
    for pod in get_pods(namespace):
        phase = pod["status"]["phase"]
        name = pod["metadata"]["name"]
        if phase not in ("Running", "Succeeded"):
            unhealthy.append((name, phase))

    if unhealthy:
        for name, phase in unhealthy:
            logging.warning(f"unhealthy pod: {name} ({phase})")
        sys.exit(1)

    logging.info(f"all pods healthy in {namespace}")
    sys.exit(0)

if __name__ == "__main__":
    main()

2. Disk Cleanup Automation

#!/usr/bin/env python3
# Deletes files older than N days under a directory, logging what was removed.
import argparse, logging, time
from pathlib import Path

logging.basicConfig(level=logging.INFO, format="%(asctime)s %(levelname)s %(message)s")

def cleanup(directory, days, pattern, dry_run):
    cutoff = time.time() - days * 86400
    freed_bytes = 0
    for path in Path(directory).rglob(pattern):
        if path.is_file() and path.stat().st_mtime < cutoff:
            size = path.stat().st_size
            if dry_run:
                logging.info(f"[dry-run] would delete {path} ({size} bytes)")
            else:
                path.unlink()
                logging.info(f"deleted {path} ({size} bytes)")
            freed_bytes += size
    logging.info(f"total freed: {freed_bytes / 1024 / 1024:.2f} MB")

if __name__ == "__main__":
    parser = argparse.ArgumentParser()
    parser.add_argument("directory")
    parser.add_argument("--days", type=int, default=7)
    parser.add_argument("--pattern", default="*.log")
    parser.add_argument("--dry-run", action="store_true")
    args = parser.parse_args()
    cleanup(args.directory, args.days, args.pattern, args.dry_run)

3. SSL Certificate Expiry Checker

#!/usr/bin/env python3
# Warns if a domain's TLS cert expires within N days.
import ssl, socket, logging
from datetime import datetime

logging.basicConfig(level=logging.INFO, format="%(asctime)s %(levelname)s %(message)s")

def days_until_expiry(hostname, port=443):
    ctx = ssl.create_default_context()
    with socket.create_connection((hostname, port), timeout=5) as sock:
        with ctx.wrap_socket(sock, server_hostname=hostname) as ssock:
            cert = ssock.getpeercert()
    expires = datetime.strptime(cert["notAfter"], "%b %d %H:%M:%S %Y %Z")
    return (expires - datetime.utcnow()).days

def check_domains(domains, warn_days=14):
    for domain in domains:
        try:
            days_left = days_until_expiry(domain)
            level = logging.WARNING if days_left < warn_days else logging.INFO
            logging.log(level, f"{domain}: {days_left} days until cert expiry")
        except Exception as e:
            logging.error(f"{domain}: could not check — {e}")

if __name__ == "__main__":
    check_domains(["example.com", "api.example.com"])

4. Log Analyzer — Error Rate Summary

#!/usr/bin/env python3
# Summarizes error/warn rate from a log file over the last N lines.
import argparse
from collections import Counter

def analyze(path, tail=10000):
    with open(path) as f:
        lines = f.readlines()[-tail:]

    levels = Counter()
    for line in lines:
        for level in ("ERROR", "WARN", "INFO"):
            if level in line:
                levels[level] += 1
                break

    total = sum(levels.values()) or 1
    print(f"Analyzed {len(lines)} lines:")
    for level, count in levels.most_common():
        print(f"  {level:6s}: {count:6d}  ({100*count/total:.1f}%)")

if __name__ == "__main__":
    parser = argparse.ArgumentParser()
    parser.add_argument("logfile")
    parser.add_argument("--tail", type=int, default=10000)
    args = parser.parse_args()
    analyze(args.logfile, args.tail)

def send_slack_alert(webhook_url, title, message, severity="warning"):

        logging.info("alert sent")

6. Daily Report Generator

#!/usr/bin/env python3
# Combines several checks into a single daily ops report, written to a file.
from datetime import datetime

def generate_report(checks: dict, output_path="daily_report.txt"):
    lines = [f"Daily SRE Report — {datetime.now():%Y-%m-%d %H:%M}", "=" * 40]
    for name, result in checks.items():
        status = "PASS" if result else "FAIL"
        lines.append(f"{name:<30s} {status}")
    lines.append("=" * 40)
    passed = sum(1 for v in checks.values() if v)
    lines.append(f"{passed}/{len(checks)} checks passed")

    report = "\n".join(lines)
    with open(output_path, "w") as f:
        f.write(report)
    print(report)
    return all(checks.values())

if __name__ == "__main__":
    results = {
        "disk usage < 85%": True,
        "all pods running": True,
        "SSL certs valid": False,
        "API health check": True,
    }
    ok = generate_report(results)
    raise SystemExit(0 if ok else 1)

Closing Note

How to Use This Handbook

This handbook is deliberately scoped to what actually gets asked in Senior SRE, DevOps, Platform Engineering, and Cloud interviews: scripting fundamentals, file and log processing, system automation, API/CLI integration, and real production scripts — not competitive-programming algorithms. Interviewers in this space are checking whether you can automate an operational task cleanly, handle failure modes, and write something a teammate could read at 3 a.m. during an incident.

Suggested path: work Chapters 1–10 (fundamentals through exception handling) if you need a refresher on core Python; if you are already fluent, skim those and focus on Chapters 9–20, which is where SRE-specific value concentrates — file/log handling, subprocess, requests, regex, and the automation script chapters. Chapter 19 is a rapid-fire bank for warm-up drills the day before an interview; Chapter 20 is the portfolio chapter — scripts worth actually having on GitHub.

Interview framing that works: when asked to write a script live, narrate your plan in one sentence, handle the obvious failure case (file missing, API timeout, empty input) without being asked, and mention what you'd add for production (logging, retries, argparse) even if you don't have time to write it. That signals SRE judgment, not just Python syntax.
