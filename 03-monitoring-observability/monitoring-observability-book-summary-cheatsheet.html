① Foundations

1 Monitoring vs Observability

Monitoring = is something wrong (known failures). Observability = why (unknowns).

Metricsdetect — symptom
Traceslocate — where
Logsexplain — evidence

white-box (internal) vs black-box (external probe).

2 Metric Types

Counterup only (requests) → rate()
Gaugeup/down (memory)
Histogrambuckets → P95/P99, aggregates
Summaryclient quantiles, hard to aggregate

Cardinality = labels × values; keep low (kills Prometheus).

② Prometheus & PromQL

3 Prometheus Architecture

Pull model: scrapes /metrics → TSDB. WAL = crash recovery before blocks.

  • Service discovery (k8s/EC2/etc.)
  • PushGateway — short-lived jobs only
  • Retention ~15–30d local; recording + alert rules
  • Operator: ServiceMonitor (services), PodMonitor (pods)

4·16 PromQL Essentials

# req/sec — Counter only! rate(reqs_total[5m]) # P95 latency histogram_quantile(.95, sum(rate(..._bucket[5m]))by(le)) # error rate sum(rate(...{status=~"5.."}))/sum(rate(...))

rate=stable/alerts · irate=reactive · increase=total. Instant vs range vector.

17 Advanced PromQL

  • topk(10, …) — hotspots
  • predict_linear(fs[6h],86400) — forecast disk full
  • Vector matching: on() / ignoring() / group_left
  • label_replace / label_join — fix labels
  • Recording rules — precompute expensive queries

rate() only on counters; aggregate replicas with sum by().

③ Grafana & Alerting

5·19 Grafana

Does NOT store metrics — queries Prometheus/Loki/Tempo.

  • Variables = one dashboard, all clusters
  • Annotations overlay deploys on graphs
  • Transformations · library panels · drill-down
  • Exec = business KPIs · Ops = technical
  • Slow dashboard = expensive queries (use recording rules)

⑤ OpenSearch & Elasticsearch

21 Node Roles & Health

Datastore + search shards
Ingestpreprocess docs

21 Shards, Heap, Backup

  • Shards = partition (scale) · Replicas = HA + read
  • ILM: Hot→Warm→Cold→Frozen (cost tiers)
  • JVM heap ≤31 GB (compressed OOPs), ≤50% RAM
  • Snapshot = backup; replica ≠ backup
  • Query: filters, no leading wildcards
⑥ Distributed Tracing & OpenTelemetry

9·22 Tracing Concepts

  • Trace = whole request · Span = one operation
  • Trace ID shared across services · Span ID per span
  • Parent/child spans; root = first span
  • Context propagation via W3C traceparent header

10·22 OpenTelemetry

SDK→OTLP→Collector→Tempo/Jaeger

Collector = receive → process → export (decouples app from backend, batch/sample/enrich).

Auto vs manual instrumentation. Vendor-neutral, one pipeline → many backends.

22 Backends & Sampling

Jaegerrich UI, mature
Tempocheap, object storage, Grafana-native

Tail sampling = decide after request → keep slow/failed (prod). Head/probabilistic = simpler.

Correlate: metrics detect → traces locate → logs explain.

⑦ SLO, Error Budgets & Availability Math

11·23 SLI / SLO / SLA

  • SLI measures (actual 99.96%)
  • SLO targets (goal 99.9%)
  • SLA promises (contract + penalty)
  • Error budget = 100% − SLO; burn rate = spend speed
  • Multi-window burn alerts; don't use CPU as an SLO

Signals: L-T-E-S · RED (services) · USE (resources).

12 Availability Math — the nines

SLODowntime/mo
99%~7h 18m
99.9%~43m
99.99%~4m 23s
99.999%~26s

avail = success / total.

12 MTxx & HA

MTBFbetween failures
MTTDto detect
MTTAto acknowledge
MTTRto recover

HA ≠ fault tolerance (FT = no interruption). Kill SPOFs with redundancy.

⑧ Incident Response, On-Call & Chaos

13·24 Incident Lifecycle

detect→SEV→IC+war room→mitigate→recover→RCA→postmortem
  • SEV1–4 by business impact (not infra)
  • IC coordinates, doesn't debug; escalate early
  • Communicate every 15–30 min (eng/exec/customer)

13·24 RCA & On-Call

  • 5 Whys / fishbone; blameless postmortem
  • CAPA = corrective + preventive actions
  • Primary/secondary, follow-the-sun; cut PagerDuty noise
  • ORR readiness review; reduce toil via automation

14 Chaos Engineering

Inject failures to prove resilience. Steady-state hypothesis + small blast radius, expand gradually.

Tools: Chaos Monkey, AWS FIS, Azure Chaos Studio, LitmusChaos, Gremlin. Game Days.

Patterns: circuit breaker · retry+backoff · timeout · bulkhead · graceful degradation.

⑨ Enterprise Architecture · DR · Capstone

15 Scaling Prometheus

Federationpull selected (aggregate)
Remote Writepush all (long-term)
Thanossidecar, HA + long-term, object storage
Mimir/Cortexcloud-scale multi-tenant TSDB

One Prometheus per cluster; aggregate globally. HA = 2 Prom + dedup.

25 DR & BCP

HAseconds · multi-AZ
DRmin–hrs · multi-region
  • RTO = downtime · RPO = data loss
  • Active-Active vs Active-Passive; DNS failover vs global LB
  • Sync (no loss) vs async (lower latency) replication
  • BCP = people+process+tech; test restores

26 Capstone & KPIs

Incident path: alert → Grafana → trace → Loki → root cause → mitigate → postmortem.

Multi-cluster (Thanos), multi-cloud; cost (cardinality/sampling); security (SSO/RBAC/encrypt/mask).

Monthly KPIs: availability, MTTD/A/R, SLO compliance, error budget, change-failure rate, deploy frequency.
