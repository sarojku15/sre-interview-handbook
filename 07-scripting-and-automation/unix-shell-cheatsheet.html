26. Script template (best practice)

#!/bin/bash
# Description : Health Check Script
# Author      : SRE
# Date        : $(date +%F)
set -euo pipefail
LOG_FILE="/var/log/health_check.log"
log() { echo "[$(date '+%F %T')] $1" | tee -a $LOG_FILE; }

check_disk() {
    usage=$(df -h / | awk 'NR==2{print $5}' | sed 's/%//')
    if [ $usage -gt 80 ]; then
        log "ERROR: Disk usage ${usage}%"
        exit 1
    else
        log "Disk usage OK: ${usage}%"
    fi
}
check_disk
log "Health check completed successfully"

27. Debugging tips

  • Run bash -x script.sh to trace execution
  • Add set -x inside a script to enable debug mode
  • Echo variable state: echo "Debug: var=$var"
  • Log everything — stdout and to a file
  • Validate inputs before acting on them
  • Test with small/synthetic data first
  • Run shellcheck script.sh — static analysis catches quoting bugs
# Install shellcheck (Linux)
sudo apt install shellcheck
shellcheck script.sh

28. Best practices

  • Always use a shebang (#!/bin/bash)
  • Enable set -euo pipefail
  • Quote variables: "$VAR"
  • Use meaningful variable names
  • Add comments and usage messages
  • Log everything (stdout + file)
  • Handle errors and exit correctly
  • Use functions for modularity
  • Validate inputs and permissions
  • Test scripts in a safe environment first
