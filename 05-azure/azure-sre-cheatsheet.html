Azure for SRE / DevOps — complete interview reference

Core services, ports, CLI/PowerShell commands, architecture flow, and troubleshooting — matches the AZ-104 scope and the Barclays prep sheet's Azure section.

1. Resource hierarchy

Tenant (Entra ID)→ Management Groups→ Subscriptions→ Resource Groups→ Resources

A Tenant is one Entra ID directory (identity boundary). Management Groups apply policy/RBAC across many subscriptions. A Subscription is the billing + hard isolation boundary. A Resource Group is a logical container (lifecycle, RBAC, deletion boundary) — resources inside can span regions, but the RG itself has one home region for its metadata.

2. Networking — VNet

VNetIsolated virtual network, defined by a CIDR block (e.g. 10.0.0.0/16)
SubnetA slice of the VNet CIDR — not tied to a single AZ like AWS
NSGStateful firewall — priority-ordered allow/deny rules, attach to subnet or NIC
Route Table (UDR)User-defined routes overriding Azure's system routes
VNet PeeringDirect private connection between VNets — non-transitive by default
Private EndpointPrivate IP into your VNet for a PaaS service (Storage, SQL, Key Vault) — data plane never touches the public internet
Service EndpointLighter-weight than Private Endpoint — keeps traffic on the Azure backbone, but the PaaS service still has a public endpoint
Azure FirewallManaged, stateful, centralized firewall — usually sits in a hub VNet

3. Compute

VMVirtual machine — B-series burstable, D-series general, F-series compute-opt, E-series memory-opt
VMSSVirtual Machine Scale Set — identical, autoscaling VM fleet behind a LB
Availability SetSpreads VMs across fault/update domains within one datacenter
Availability ZoneSpreads resources across physically separate datacenters in a region — stronger guarantee than Availability Sets
AKSManaged Kubernetes — control plane free, you pay for node pools
App ServicePaaS for web apps/APIs — no VM/cluster management
Azure FunctionsServerless, event-driven compute
Container AppsServerless containers built on KEDA/Dapr — lighter than AKS

4. Storage

Storage AccountUmbrella for Blob, File, Queue, Table storage — globally unique name
Blob StorageObject storage — Hot, Cool, Cold, Archive tiers
Azure FilesManaged SMB/NFS file shares — mountable like a network drive
Managed DisksBlock storage for VMs — Standard HDD/SSD, Premium SSD, Ultra Disk
RedundancyLRS (local), ZRS (zone), GRS (geo), RA-GRS (geo + readable secondary)
Lifecycle managementAuto-tier/delete blobs by age — the cost-control lever

5. Identity & security

Entra IDAzure's identity provider (formerly Azure AD) — users, groups, service principals
Service PrincipalAn application's identity — used by pipelines/scripts to auth (has a client secret or cert)
Managed IdentityAzure-managed identity for a resource — no stored credentials at all. System-assigned (tied to the resource) or user-assigned (shared, reusable)
RBACRole assignments (Owner, Contributor, Reader, custom) scoped to management group / subscription / RG / resource
Key VaultStores secrets, keys, and certificates — soft-delete + purge protection for prod
Azure PolicyEnforces org rules (allowed regions, required tags, no public IPs) — the governance/compliance gate
Defender for CloudSecurity posture management + threat detection

7. High availability architecture — real production scenario

Traffic Manager / Front Door
DNS/global routing, failover→ Application Gateway
L7, WAF, TLS termination→ AKS / VMSS
multi-zone compute→ Azure SQL / Cosmos DB
zone-redundant, geo-replicated

Every tier spans Availability Zones within a region; Traffic Manager or Front Door handles cross-region failover for full DR; Application Gateway's health probes eject unhealthy backends.

8. Load balancing options

Azure Load BalancerL4 — TCP/UDP, regional or global, ultra-low latency
Application GatewayL7 — HTTP(S), path/host routing, TLS termination, WAF built in
Front DoorGlobal L7 — CDN + WAF + routing across regions, anycast entry point
Traffic ManagerDNS-based global routing/failover — no data plane, just smart DNS answers

10. Azure CLI & PowerShell — commands you'll actually be asked to run

# Login & context
az login
az account show
az account set --subscription "My Subscription"

# Resource groups & resources
az group list -o table
az resource list --resource-group my-rg -o table

# VMs
az vm list -d -o table
az vm start --resource-group my-rg --name my-vm
az vm stop --resource-group my-rg --name my-vm
az vm restart --resource-group my-rg --name my-vm

# AKS
az aks get-credentials --resource-group my-rg --name my-cluster
az aks show --resource-group my-rg --name my-cluster --query "kubernetesVersion"
az aks nodepool list --resource-group my-rg --cluster-name my-cluster -o table
az aks upgrade --resource-group my-rg --name my-cluster --kubernetes-version 1.30.2

# Storage
az storage account list -o table
az storage blob upload --account-name mystorage --container-name mycontainer --file file.txt

# Key Vault
az keyvault secret show --vault-name my-kv --name my-secret
az keyvault secret set --vault-name my-kv --name my-secret --value "supersecret"

# Monitor / logs
az monitor metrics list --resource <resource-id> --metric "Percentage CPU"
az monitor log-analytics query --workspace <workspace-id> --analytics-query "AzureActivity | take 10"

13. Troubleshooting guide

SymptomLikely cause / where to look
VM unreachable (timeout)NSG inbound rule missing (check both subnet and NIC level) · UDR misrouting traffic · no public IP assigned
VM "connection refused"Nothing listening on that port — service not running or bound to localhost only
App Gateway 502Backend pool unhealthy, backend not responding on the configured port, or backend TLS cert mismatch
App Gateway "backend health = unhealthy"Health probe path returning non-200, NSG blocking the probe, or app not listening on expected port
RBAC "does not have authorization"Role assigned at wrong scope, missing action in a custom role, or a Deny assignment / Azure Policy blocking it
Managed Identity can't reach Key VaultAccess policy or RBAC role not granted to the identity, or Key Vault firewall blocking the resource's network
AKS pod can't pull imageACR attach missing (az aks update --attach-acr) or image pull secret misconfigured
Azure SQL high DTU/CPUMissing indexes, connection pool exhaustion, or check Query Performance Insight for the offending query

14. Interview quick Q&A

Q: Azure Load Balancer vs Application Gateway — when do you pick each?
A: Load Balancer for L4 TCP/UDP at low latency; Application Gateway for L7 HTTP(S) with path/host routing and built-in WAF — the default internet-facing front door for a web app.
Q: System-assigned vs user-assigned Managed Identity?
A: System-assigned is created with and tied to one resource's lifecycle (deleted with it); user-assigned is a standalone identity that can be shared across multiple resources and outlives any one of them.
Q: How does a pod in AKS get an Azure token without stored credentials?
A: Workload Identity federates a Kubernetes service account to a Managed Identity via OIDC — the pod exchanges its K8s token for an Azure AD token, no secrets stored anywhere.
Q: NSG vs Azure Firewall — when do you need both?
A: NSGs are free, stateful, per-subnet/NIC allow-deny rules — your first line of defense. Azure Firewall is a managed, centralized appliance (often in a hub VNet) for FQDN filtering, threat intelligence, and consistent policy across many spoke VNets — used together in hub-spoke topologies.
Q: Resource Group deletion protection — how do you prevent an accidental delete?
A: Apply a CanNotDelete or ReadOnly Azure Resource Lock on the RG or resource — locks aren't RBAC, they block the operation regardless of the caller's role.
Q: How would you reduce Azure cost for a workload with a spiky, unpredictable pattern?
A: Right-size with Azure Advisor recommendations, use Reserved Instances/Savings Plans for the steady baseline, Spot VMs for fault-tolerant/batch work, and autoscale VMSS/AKS on a metric close to real load, not just CPU.
