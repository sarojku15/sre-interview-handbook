Kubernetes — Memorize-the-Book Summary

Every chapter compressed to its must-know facts, comparisons, and the numbers interviewers test. Read top to bottom; the troubleshooting and numbers blocks are the highest-yield for recall.

① Core & Control Plane

1 Fundamentals

K8s = container orchestration (not "container platform"). Core idea: desired-state reconciliation — actual is continuously driven toward desired.

Gives: self-healing, autoscaling, rolling updates/rollback, service discovery, scheduling.

1 Architecture

Control plane: API server (sole entry), etcd (state), scheduler (placement), controller-mgr (reconcile).

Worker: kubelet, kube-proxy, runtime (containerd), pods.

1 apply → Running

apply→API→authN→RBAC→admission→etcd→ctlr→RS→scheduler→kubelet→Running

Desired state is stored first, then reconciled.

2 Control Plane parts

  • API server — only one talking to etcd; authN/Z + admission
  • etcd — distributed KV, all state; odd quorum 3/5; back up
  • Scheduler — filter → score → bind
  • Controller-mgr — reconcile loops
  • kubelet — node agent, runs pods
  • kube-proxy — implements Services
② Pods & Workloads

3 Pods

Smallest unit; 1+ containers share network (same IP, localhost) + volumes.

Phases: Pending→Running→Succeeded/Failed. restartPolicy: Always(def)/OnFailure/Never.

Init containers run first, sequential; sidecars run alongside (logging, proxy).

4 ReplicaSet / Deployment

RS keeps N pods (don't create directly). Deployment manages RS → rolling updates + rollback.

maxSurgeextra pods during update
maxUnavailablepods that can be down

rollout undo deploy/<d>

5 StatefulSet

For stateful apps (DB, Kafka). Gives stable identity (pod-0..N), stable DNS, per-pod PVC via volumeClaimTemplates.

Ordered create/delete; needs a Headless Service.

6·7 DaemonSet / Jobs

DaemonSet = one pod per node (agents, log/metric collectors); auto-added on new nodes.

Job = run-to-completion: completions, parallelism, backoffLimit, activeDeadlineSeconds, ttlSecondsAfterFinished.

CronJob = scheduled Job (cron); concurrencyPolicy.

③ Networking & Exposure

8 Services

Stable virtual IP + DNS over changing pods (pod IPs are ephemeral).

ClusterIPinternal (default)
NodePortport on each node
LoadBalancercloud external LB
ExternalNameCNAME alias
Headlessno clusterIP → direct pod DNS (StatefulSet)

9 Ingress

L7 HTTP(S) routing in. Ingress = rules; Controller = implements them (NGINX…).

Host + path routing · TLS termination · IngressClass · default backend.

12·23 Networking deep

Flat network: every pod routable IP. CNI: Calico (policy/BGP), Cilium (eBPF), Flannel (simple overlay).

CoreDNS = discovery · kube-proxy iptables vs IPVS (scales better) · EndpointSlices scale endpoints.

NetworkPolicy = pod firewall (allow-all until a policy selects the pod). Service mesh = mTLS + traffic mgmt via sidecars.

④ Config & Storage

10 ConfigMaps & Secrets

ConfigMap = non-secret config (env vars or volume). Secret = base64, NOT encryption.

Enable encryption-at-rest; never put secrets in ConfigMaps. Env vars need restart; volume mounts can refresh.

11 PV / PVC / SC / CSI

StorageClass→PVC→PV→Pod

PV = volume, PVC = claim. SC = dynamic provisioning. CSI = pluggable driver.

RWO/ROX/RWX1-node / many-read / many-write
ReclaimRetain · Delete

13 Scheduling

Scheduler: filter → score → bind.

  • nodeSelector simple pin; nodeAffinity required/preferred
  • pod (anti-)affinity co-locate / spread
  • Taint (node) repels; Toleration (pod) allows — NoSchedule/PreferNoSchedule/NoExecute
  • PriorityClass → preemption
⑤ Resources · Probes · Autoscaling

14 Resources & QoS

requests = scheduling reservation; limits = hard cap.

QoS: Guaranteed (req=lim) > Burstable > BestEffort (evicted first).

CPU over limit → throttle; memory over limit → OOMKilled. Node pressure → eviction.

15 Probes

livenessfail → restart pod
readinessfail → drop from endpoints
startupgate for slow boot

Types: httpGet · tcpSocket · exec. Aggressive liveness → restart loops.

16 Autoscaling

HPApod count (CPU/mem/custom)
VPArequests/limits
Cluster ASnode count (unschedulable pods)

Prefer horizontal scaling in production.

⑥ Security · Observability · Release

17 Security

  • RBAC — Role/ClusterRole + Binding; least privilege
  • ServiceAccount = pod identity
  • Pod Security Standards — privileged/baseline/restricted
  • OPA Gatekeeper / Kyverno policy · Falco runtime · image scanning · encrypt secrets at rest

18 Observability

Prometheus (pull metrics) → Grafana (dashboards) + Alertmanager (routing).

Exporters: node-exporter, kube-state-metrics, cAdvisor. Logs: Fluent Bit DaemonSet → Loki/ELK (stdout/stderr).

Golden signals: latency, traffic, errors, saturation.

19 Release Strategies

Rollinggradual, zero-downtime (default)
Recreatestop all, start new
Blue-Greenflip traffic, instant rollback
Canarysmall %, watch, expand

Progressive delivery: Argo Rollouts / Flagger.

⑦ Backup/DR · Performance · GitOps

21 Backup & DR

  • etcd critical → snapshot regularly (etcdctl snapshot save)
  • Velero = cluster + PV backup/restore + migration
  • HA: 3/5 etcd quorum, multi-AZ nodes, multi-master
  • RTO = recovery time · RPO = data loss; test restores

24 Performance

  • CPU throttle (CFS) vs OOMKilled — different fixes
  • Right-size requests/limits from history
  • Small images: multi-stage, distroless/alpine
  • etcd on SSD + defrag; watch API-server latency
  • JVM: MaxRAMPercentage

25 GitOps / ArgoCD

Git = source of truth; controller reconciles live → desired.

  • sync (auto/manual) · selfHeal reverts drift · prune deletes orphans
  • App vs ApplicationSet (multi-cluster) · app-of-apps
  • Helm/Kustomize · sealed/external secrets

⚡ Numbers & one-liners to lock in

Core idea — desired-state reconciliation.
API server — only component that talks to etcd.
etcd quorum — odd numbers, 3 or 5.
CPU vs Mem — CPU over-limit throttles; memory over-limit is OOMKilled.
QoS evict order — BestEffort → Burstable → Guaranteed.
Probes — liveness restarts, readiness gates traffic, startup for slow boot.
Secret — base64, not encryption (enable encryption at rest).
Headless Service — gives StatefulSet stable per-pod DNS.
Ingress — rules need a Controller to do anything.
kube-proxy — IPVS scales better than iptables.
DaemonSet — one pod per node, auto on new nodes.
maxSurge / maxUnavailable — tune rolling updates.
NetworkPolicy — allow-all until a policy selects the pod.
Velero = backups; etcd snapshot = cluster state. Replicas ≠ backups.
GitOps — Git is truth; selfHeal kills drift, prune kills orphans.
